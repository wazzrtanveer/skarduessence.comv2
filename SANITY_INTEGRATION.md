# Sanity CMS Integration Guide

This document summarizes the integration of Sanity CMS into the Skardu Essence Astro project.

## 🚀 Overview
The project has been migrated from a static JSON-based product management system (`src/data/products.js`) to a headless CMS using **Sanity**. This allows for non-technical management of products, pricing, and assets via a visual dashboard.

## 🏗️ Technical Architecture
- **Framework:** Astro (Static mode)
- **CMS:** Sanity (Hosted Content Lake)
- **Dashboard:** Embedded Sanity Studio at `/admin`
- **Frontend Integration:** `@sanity/astro` + `@sanity/client`
- **Styling:** Tailwind CSS (v4)
- **Deployment:** Cloudflare Pages (Free tier)

## 📂 Project Structure Changes
- `sanity.config.js`: Main configuration for the embedded Sanity Studio.
- `src/sanity/schemaTypes/`: Contains the Content Model (Schemas).
  - `product.js`: Defines the Product fields (Name, Price, Category, etc.).
- `src/sanity/lib/client.js`: Utility for fetching data and generating optimized image URLs.
- `.env`: Contains Project ID and Dataset information.

## 📝 Product Schema
The `product` document type includes:
- **Name:** String (required)
- **Slug:** Auto-generated from name
- **Description:** Text area
- **Price:** Number (PKR)
- **Image:** Sanity Image with Hotspot support
- **Category:** Dropdown (Supplements, Food, Snacks, Oils, Spices)
- **Featured:** Boolean toggle (for homepage slider)

## 🔄 Data Fetching & Fallback Logic
The Home, Shop, and Product detail pages use a **Hybrid Fetching Strategy**:
1. The page attempts to fetch live data from Sanity.
2. If Sanity returns no products (e.g., first setup), the site automatically falls back to the local `src/data/products.js` file.
3. This ensures the website is never broken or empty during the transition.

## 🛠️ Management Instructions

### Local Development
1. Run `npm run dev`.
2. Visit `http://localhost:4321/admin`.
3. Log in with your Sanity account.

### Adding Products
- Go to the **Product** tab.
- Click the **+** icon.
- **Image Tip:** Use the "Hotspot" tool in Sanity to ensure the product stays centered regardless of the screen size.
- **Publish:** Changes are saved to the database immediately, but a site rebuild is required to see them on the live URL.

## ☁️ Cloudflare Pages Deployment Guide

To make your Sanity integration work on the live site, follow these steps:

### 1. Set Environment Variables
In your Cloudflare Pages Dashboard:
- Go to **Settings** > **Environment variables**.
- Add the following variables for both **Production** and **Preview**:
  - `PUBLIC_SANITY_PROJECT_ID`: `8qohnu41`
  - `PUBLIC_SANITY_DATASET`: `production`

### 2. Configure CORS in Sanity
You must allow your live website to talk to Sanity:
- Go to [manage.sanity.io](https://manage.sanity.io)
- Select your project > **API** > **CORS origins**.
- Click **Add CORS origin**.
- Enter your live URL (e.g., `https://skarduessence.com` and `https://*.pages.dev`).
- **CRITICAL:** Check the box **"Allow credentials"**.

### 3. Setup Automatic Updates (Webhooks)
To make your site update automatically when you hit "Publish" in Sanity:
- **In Cloudflare:** Go to **Settings** > **Builds & deployments** > **Deploy hooks**.
- Create a new hook (e.g., named "Sanity Update") and copy the URL.
- **In Sanity:** Go to [manage.sanity.io](https://manage.sanity.io) > **API** > **Webhooks**.
- Click **Create Webhook**.
- Name: `Cloudflare Deploy`
- URL: Paste your Cloudflare Deploy Hook URL.
- Dataset: `production`
- Trigger on: `Create`, `Update`, `Delete`.
- Filter: `_type in ["product", "homePage", "aboutPage", "settings"]`

### 4. Build Command
Ensure your build command in Cloudflare is set to:
`npm run build`

## 🧪 Troubleshooting
- **White Screen in Admin:** Ensure the `PUBLIC_` variables are correctly set in Cloudflare.
- **Data not updating:** Check the Sanity Webhook logs to see if the trigger reached Cloudflare.
- **NoMatchingRenderer Error:** Ensure `@astrojs/react` is installed and listed in `astro.config.mjs`.
